## 2. 数据库部署

### 2.1 安装MySQL

#### Windows环境
1. 下载MySQL安装包：https://dev.mysql.com/downloads/mysql/
2. 运行安装程序，按照向导完成安装
3. 设置root密码（建议使用强密码）
4. 启动MySQL服务

#### Linux环境（CentOS）
```bash
# 下载MySQL Yum Repository
wget https://dev.mysql.com/get/mysql80-community-release-el7-3.noarch.rpm

# 安装Repository
rpm -ivh mysql80-community-release-el7-3.noarch.rpm

# 安装MySQL
yum install mysql-server -y

# 启动MySQL服务
systemctl start mysqld
systemctl enable mysqld

# 获取临时密码
grep 'temporary password' /var/log/mysqld.log

# 登录MySQL并修改密码
mysql -uroot -p
ALTER USER 'root'@'localhost' IDENTIFIED BY 'YourPassword';
```

### 2.2 创建数据库

```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS maple_edu DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;

-- 使用数据库
USE maple_edu;

-- 创建用户（可选）
CREATE USER 'ruoyi'@'%' IDENTIFIED BY 'Ruoyi@123456';

-- 授权
GRANT ALL PRIVILEGES ON maple_edu.* TO 'ruoyi'@'%';

-- 刷新权限
FLUSH PRIVILEGES;
```

### 2.3 导入数据库脚本

1. 找到项目中的SQL脚本文件（通常在项目根目录或sql文件夹）
2. 使用以下命令导入：

```bash
# Windows
mysql -uroot -p maple_edu < maple_edu.sql

# Linux
mysql -uroot -p maple_edu < maple_edu.sql
```

或使用MySQL客户端工具（如Navicat、MySQL Workbench）导入SQL文件。

## 3. 项目配置

### 3.1 修改数据库配置

编辑 `ruoyi-admin/src/main/resources/application-druid.yml` 文件：

```yaml
spring:
    datasource:
        type: com.alibaba.druid.pool.DruidDataSource
        driverClassName: com.mysql.cj.jdbc.Driver
        druid:
            master:
                # 修改数据库地址
                url: jdbc:mysql://localhost:3306/maple_edu?useUnicode=true&characterEncoding=utf8&zeroDateTimeBehavior=convertToNull&useSSL=true&serverTimezone=GMT%2B8

                username: root
                # 修改数据库密码
                password: YourPassword
```

### 3.2 修改应用配置

编辑 `ruoyi-admin/src/main/resources/application.yml` 文件：

```yaml
# 项目相关配置
ruoyi:
  # 名称
  name: Qianjiaming

  version: 1

  # 服务器的HTTP端口，默认为80
```

## 5. 部署方式

### 5.1 方式一：直接运行Jar包（推荐）

#### Windows环境
```cmd
# 进入jar包所在目录
cd ruoyi-admin\target

# 运行jar包
java -jar ruoyi-admin.jar

# 或指定配置文件运行
java -jar ruoyi-admin.jar --spring.config.location=../resources/application.yml
```

#### Linux环境
```bash
# 进入jar包所在目录
cd /path/to/project/ruoyi-admin/target

# 后台运行jar包
nohup java -jar ruoyi-admin.jar > /dev/null 2>&1 &

# 或使用screen运行
screen -S ruoyi
java -jar ruoyi-admin.jar
# 按Ctrl+A+D退出screen

# 查看运行日志
tail -f logs/sys-info.log
```

### 5.2 方式二：使用Tomcat部署

#### 准备工作
1. 下载并安装Tomcat：https://tomcat.apache.org/
2. 修改 `pom.xml` 中的打包方式为war

#### 修改pom.xml
```xml
<packaging>war</packaging>
```

#### 打包
```bash
mvn clean package -DskipTests
```

#### 部署
```bash
# 将war包复制到Tomcat的webapps目录
cp ruoyi-admin/target/ruoyi-admin.war /path/to/tomcat/webapps/

# 启动Tomcat
cd /path/to/tomcat/bin
./startup.sh
```

### 5.3 方式三：使用Docker部署（推荐）

#### 创建Dockerfile

在项目根目录创建 `Dockerfile`：

```dockerfile
FROM openjdk:8-jdk-alpine

VOLUME /tmp

COPY ruoyi-admin/target/ruoyi-admin.jar app.jar

ENTRYPOINT ["java","-jar","/app.jar"]

EXPOSE 8888
```

#### 构建Docker镜像
```bash
# 构建镜像
docker build -t ruoyi-edu:4.7.9 .

# 查看镜像
docker images
```

#### 运行Docker容器
```bash
# 运行容器
docker run -d \
  --name ruoyi-edu \
  -p 8888:8888 \
  -v /home/ruoyi/uploadPath:/home/ruoyi/uploadPath \
  -e SPRING_DATASOURCE_URL="jdbc:mysql://host.docker.internal:3306/maple_edu?useUnicode=true&characterEncoding=utf8&zeroDateTimeBehavior=convertToNull&useSSL=true&serverTimezone=GMT%2B8" \
  -e SPRING_DATASOURCE_USERNAME="root" \
  -e SPRING_DATASOURCE_PASSWORD="YourPassword" \
  ruoyi-edu:4.7.9

# 查看容器日志
docker logs -f ruoyi-edu

# 停止容器
docker stop ruoyi-edu

# 启动容器
docker start ruoyi-edu

# 删除容器
docker rm -f ruoyi-edu
```

#### 使用Docker Compose（推荐）

创建 `docker-compose.yml`：

```yaml
version: '3.8'

services:
  mysql:
    image: mysql:8.0
    container_name: ruoyi-mysql
    environment:
      MYSQL_ROOT_PASSWORD: 123456
      MYSQL_DATABASE: maple_edu
    ports:
      - "3306:3306"
    volumes:
      - mysql-data:/var/lib/mysql
      - ./maple_edu.sql:/docker-entrypoint-initdb.d/maple_edu.sql
    command: --default-authentication-plugin=mysql_native_password

  ruoyi:
    image: ruoyi-edu:4.7.9
    container_name: ruoyi-edu
    ports:
      - "8888:8888"
    volumes:
      - /home/ruoyi/uploadPath:/home/ruoyi/uploadPath
      - ./logs:/logs
    environment:
      SPRING_DATASOURCE_URL: "jdbc:mysql://mysql:3306/maple_edu?useUnicode=true&characterEncoding=utf8&zeroDateTimeBehavior=convertToNull&useSSL=false&serverTimezone=GMT%2B8"
      SPRING_DATASOURCE_USERNAME: "root"
      SPRING_DATASOURCE_PASSWORD: "123456"
    depends_on:
      - mysql
    restart: always

volumes:
  mysql-data:
```

启动服务：
```bash
# 启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f ruoyi

# 停止所有服务
docker-compose down

# 停止并删除数据卷
docker-compose down -v
```

## 7. Nginx反向代理配置（可选）

### 7.1 安装Nginx

#### Linux环境（CentOS）
```bash
# 安装EPEL源
yum install epel-release -y

# 安装Nginx
yum install nginx -y

# 启动Nginx
systemctl start nginx
systemctl enable nginx
```

### 7.2 配置Nginx

编辑 `/etc/nginx/conf.d/ruoyi-edu.conf`：

```nginx
upstream ruoyi_edu {
    server 127.0.0.1:8888 weight=1 max_fails=2 fail_timeout=30s;
}

server {
    listen 80;
    server_name your-domain.com;

    # 访问日志
    access_log /var/log/nginx/ruoyi-edu-access.log;
    # 错误日志
    error_log /var/log/nginx/ruoyi-edu-error.log;

    # 客户端上传文件大小限制
    client_max_body_size 100m;

    # 代理配置
    location / {
        proxy_pass http://ruoyi_edu;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # WebSocket支持
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }

    # 静态资源缓存
    location ~* \.(jpg|jpeg|png|gif|ico|css|js|svg|woff|woff2|ttf|eot)$ {
        proxy_pass http://ruoyi_edu;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
}
```

测试并重启Nginx：
```bash
# 测试配置
nginx -t

# 重启Nginx
systemctl restart nginx
```

## 9. 常见问题

### 9.1 端口被占用

**问题**：启动时提示端口8888被占用

**解决方案**：
```bash
# 查看端口占用
netstat -tunlp | grep 8888

# 杀死占用进程
kill -9 <pid>

# 或修改application.yml中的端口配置
server:
  port: 8080
```

### 9.2 数据库连接失败

**问题**：无法连接到数据库

**解决方案**：
1. 检查MySQL服务是否启动
2. 检查数据库地址、端口、用户名、密码是否正确
3. 检查数据库用户权限
4. 检查防火墙设置

### 9.3 文件上传失败

**问题**：上传文件时提示失败

**解决方案**：
1. 检查文件上传路径是否存在
2. 检查文件上传路径的读写权限
3. 检查文件大小是否超过限制（默认10MB）

### 9.4 内存不足

**问题**：系统运行缓慢或崩溃

**解决方案**：
```bash
# 增加JVM内存
java -Xms512m -Xmx1024m -jar ruoyi-admin.jar
```

### 9.5 日志文件过大

**问题**：日志文件占用过多磁盘空间

**解决方案**：
1. 配置日志滚动策略
2. 定期清理历史日志
3. 使用logback配置文件控制日志级别

### 9.6 验证码显示异常

**问题**：验证码无法显示

**解决方案**：
1. 检查Kaptcha配置
2. 检查字体文件是否存在
3. 检查浏览器缓存

## 12. 备份与恢复

### 12.1 数据库备份

```bash
# 备份数据库
mysqldump -uroot -p maple_edu > maple_edu_backup_$(date +%Y%m%d).sql

# 恢复数据库
mysql -uroot -p maple_edu < maple_edu_backup_20251230.sql
```

### 12.2 自动备份脚本

创建备份脚本 `/home/ruoyi/backup.sh`：

```bash
#!/bin/bash

# 备份目录
BACKUP_DIR="/home/ruoyi/backup"
# 数据库信息
DB_NAME="maple_edu"
DB_USER="root"
DB_PASS="YourPassword"

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份数据库
mysqldump -u$DB_USER -p$DB_PASS $DB_NAME > $BACKUP_DIR/$DB_NAME_$(date +%Y%m%d_%H%M%S).sql

# 删除7天前的备份
find $BACKUP_DIR -name "*.sql" -mtime +7 -delete

echo "Backup completed at $(date)"
```

设置定时任务：
```bash
# 编辑crontab
crontab -e

# 每天凌晨2点执行备份
0 2 * * * /home/ruoyi/backup.sh >> /home/ruoyi/backup.log 2>&1
```

### D. 常用命令速查

```bash
# Maven打包
mvn clean package -DskipTests

# 启动服务
java -jar ruoyi-admin.jar

# 后台运行
nohup java -jar ruoyi-admin.jar > /dev/null 2>&1 &

# 查看日志
tail -f logs/sys-info.log

# 查看进程
ps -ef | grep java

# 停止进程
kill -9 <pid>

# 数据库备份
mysqldump -uroot -p maple_edu > backup.sql

# 数据库恢复
mysql -uroot -p maple_edu < backup.sql
```
